# Prompt Engineering

Best Practices und Techniken für effektive Prompts mit Claude.

## Was ist Prompt Engineering?

Die Kunst, klare, präzise Anweisungen zu schreiben, die Claude hilfreiche, genaue Antworten geben.

## Grundprinzipien

1. **Klarheit** — Genau aussprechen, was du brauchst
2. **Kontext** — Relevante Informationen geben
3. **Format** — Gewünsches Output-Format angeben
4. **Beispiele** — Few-shot Examples geben, wenn möglich
5. **Iteration** — Feedback geben und verfeinern

## Techniken für Entwicklung

### Rückfragen einfordern
```
"Stell mir Rückfragen, bis du dir absolut sicher bist, was ich will — fang erst dann an."
```

### Offene Fragen stellen
Du beschreibst das Ziel — Claude bekommt die Denkarbeit zurück.
```
"Wie sollte ein Kontaktformular aussehen, damit möglichst viele Leute es ausfüllen?"
```

### Chain of Thought
```
"Erklär dein Denken Schritt für Schritt, bevor du Code schreibst."
```

### System Prompt
```
"Du bist ein erfahrener Software Architect. Denk in SOLID-Prinzipien."
```

### Kontext einbinden
```
"Nutze diese bestehende Codebase als Referenz..."
```

## Best Practices für meine Arbeit

- Architektur-Anforderungen klarmachen
- SOLID-Prinzipien explizit erwähnen
- Code-Style und Conventions vorgeben
- Sicherheits- und Performance-Anforderungen nennen
